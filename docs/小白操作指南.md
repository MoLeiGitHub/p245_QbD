# QbD Beta 小白操作指南

## 1. 启动系统

```bash
docker compose up --build
```

访问地址：

- 前端: http://localhost:18102
- API: http://localhost:18101

## 2. 登录账号

- Owner: `owner@example.com / owner123`
- Editor: `editor@example.com / editor123`
- Reviewer: `reviewer@example.com / reviewer123`
- Viewer: `viewer@example.com / viewer123`

建议先用 Owner。

## 3. 完整流程（一次跑通）

1. 新建项目。
2. 在 `Study 向导` 创建 Study：
   - 新手建议先选 `full_factorial`。
   - 因子和响应直接在表格里填，不需要写 JSON。
3. 点击 `生成 DOE`。
4. 在 `结果导入` 里点击 `下载当前 Study 导入模板 CSV`。
5. 在模板里填写每个 `run_order` 的响应值，保存后上传。
6. 点击 `导入结果`。
7. 点击 `运行分析`，查看 ANOVA 与图表。
8. 在 `Design Space` 选择 X/Y 因子，固定其他因子后点击生成。
   - 可查看 `Overlay（通过/失败）`、`3D响应面`、`2D等高线`。
9. 点击 `更新风险`、`生成控制策略`。
10. 在 `报告与审核` 中提交/批准并导出 PDF。

## 4. full_factorial 示例

示例结果文件：

- `docs/sample_full_factorial_results.csv`

表头格式必须包含：

```csv
run_order,Dissolution,CU_RSD,Hardness
```

## 5. Design Space 固定因子示例

如果 X 因子是 `A_PSD`，Y 因子是 `B_Disintegrant`，那就固定第三个因子 `C_MCC`。

系统界面已改为直接输入数值，不用再写 JSON。

常用值：

- `C_MCC = 50`
- 或 `C_MCC = 33.3`
- 或 `C_MCC = 66.7`

## 6. 常见问题

### 6.1 运行分析失败，但容器日志没有明显报错
这通常是分析异常被捕获并写进任务状态。现在界面会直接显示失败原因。

### 6.2 imported_runs = 0
导入文件的 `run_order`/`run_id` 没匹配当前 Study 的 DOE。请重新下载模板再填数据。

### 6.3 mixture_2comp 生成 DOE 报错
该设计必须且只能 2 个因子。

## 7. 升级代码会不会丢项目数据
一般不会。数据库数据在 Docker volume（`pg_data`）里。

会丢数据的情况：

- 执行了 `docker compose down -v`
- 手动删除了 Docker volume
- 使用了不同 compose 项目名导致新 volume
